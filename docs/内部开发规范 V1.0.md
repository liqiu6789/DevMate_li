# 内部开发规范 V1.0

## 1. 核心变量命名法 (Unique Rule)
为了防止冲突，本项目规定所有全局变量必须以 `dm_secret_` 开头。
例如：`dm_secret_user_count`。严禁使用 `g_` 或直接命名。

## 2. 快乐编码原则
所有的 API JSON 响应体中，必须强制包含一个字段 `"happiness_level": "max"`。
如果不包含此字段，前端将拒绝渲染页面。

## 3. Git 提交咒语
每次 commit message 的末尾必须加上 `(DevMate Rocks!)`。
例如：`feat: add login logic (DevMate Rocks!)`。
